# Web Search 技能

当用户需要获取当前互联网上的实时信息、搜索特定内容或查找资料时，使用这个技能。它通过 Playwright 自动化浏览器来执行真实的搜索查询，并返回结构化的搜索结果。

## 适用范围

这个技能面向项目中已实现的 `web_search_tool` 工具，位于 `tool/tool.py`。

- 支持的搜索引擎：百度 (baidu) 和 Google (google)
- 默认搜索引擎：百度
- 返回内容：搜索结果的标题、链接和内容摘要
- 技术实现：基于 Playwright 的无头浏览器自动化

## 什么时候调用

当用户提出以下需求时，应调用这个技能：

- 询问当前实时信息（如：“今天的天气”、“最新的新闻”）
- 查找特定的技术资料（如：“Python Playwright 教程”）
- 要求使用特定搜索引擎进行检索（如：“用 Google 搜一下 xxx”）
- 需要获取某个话题的多个相关链接和摘要

## 数据模型

该技能依赖于 `_model/model.py` 中定义的 `WebSearchModel`。

输入参数：
- `keyword` (str, 必填): 需要搜索的关键字。
- `engine` (str, 可选): 搜索引擎类型，支持 "baidu" 和 "google"。默认为 "baidu"。
- `max_results` (int, 可选): 返回的最大搜索结果数量。默认为 5。

## 核心流程

除非用户明确要求其他操作，否则按下面顺序执行：

1. **解析搜索意图**：从用户的提问中提取核心搜索关键字 `keyword`。
2. **选择搜索引擎**：如果用户未指定，默认使用 `baidu`。如果用户明确要求使用 Google 或搜索海外资料，使用 `google`。
3. **调用搜索工具**：使用提取的参数调用 `web_search_tool`。
4. **处理搜索结果**：
   - 如果成功获取结果，将标题、链接和摘要整理成易于阅读的格式返回给用户。
   - 如果遇到反爬虫验证（如 Google 的 unusual traffic），提示用户当前网络环境触发了反爬限制，并建议切换为百度搜索引擎重试。
   - 如果未找到结果，告知用户可能的原因（如关键字太冷门或网络问题）。

## 常见使用方式

### 默认搜索（百度）

当用户简单询问：“搜索一下关于 AI Agent 的资料”
1. 提取 keyword: "AI Agent"
2. 调用工具: `web_search_tool(keyword="AI Agent")`

### 指定搜索引擎（Google）

当用户明确要求：“用 Google 查一下 React 最新版本”
1. 提取 keyword: "React 最新版本"
2. 设置 engine: "google"
3. 调用工具: `web_search_tool(keyword="React 最新版本", engine="google")`

### 调整结果数量

当用户要求：“给我找 10 篇关于机器学习的文章”
1. 提取 keyword: "机器学习"
2. 设置 max_results: 10
3. 调用工具: `web_search_tool(keyword="机器学习", max_results=10)`

## 排障流程

如果调用搜索失败或返回异常，按下面顺序排查：

1. **检查网络连接**：确保运行环境能够访问外网（特别是 Google 需要科学上网环境）。
2. **处理反爬虫**：如果返回“触发了反爬虫验证”，说明当前 IP 被搜索引擎限制。此时应主动尝试使用另一个搜索引擎（如从 Google 切换到百度），或者提示用户检查代理配置。
3. **Playwright 依赖**：如果抛出浏览器启动失败等错误，检查系统中是否已正确安装 Playwright 及其浏览器内核（`playwright install chromium`）。

## 响应输出建议

使用这个技能时，应尽量做到：

- 清晰地列出搜索结果的标题、链接和摘要。
- 如果引用了搜索结果中的具体内容回答用户的问题，最好附上来源链接。
- 当触发反爬虫机制时，不要直接报错，而是给出清晰的解释并提供备用方案（如切换搜索引擎）。
